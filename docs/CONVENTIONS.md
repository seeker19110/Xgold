# CONVENTIONS — Sổ quy ước dự án Xgold

> Mỗi pattern lặp lại có MỘT cách đúng duy nhất. Tính năng mới & mọi lần sửa phải đối chiếu.
> Lệch quy ước = phát hiện Nhóm 12 khi audit. Đổi quy ước lớn → ghi ADR.
> Lập bằng cách đọc code thật ngày 2026-07-03 (không đoán).

| Pattern                             | Cách đúng duy nhất                                                                                                                                                              | File ví dụ chuẩn                                                                                       | Ghi chú                                                                                                                                                                                                     |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Validate đầu vào                    | Zod schema tại ranh giới: query param API (`safeParse`), response provider ngoài, cấu hình từ URL/localStorage                                                                  | `app/api/candles/route.ts`, `lib/indicators/config.ts`                                                 | Input hỏng → trả lỗi có cấu trúc hoặc rơi về giá trị mặc định an toàn, không crash.                                                                                                                         |
| Dạng lỗi API route                  | `NextResponse.json({ error: <string \| ZodError.flatten()> }, { status })` — 400 (input sai), 404 (không tìm thấy), 500 (lỗi hạ tầng)                                           | `app/api/candles/route.ts`, `app/api/domestic-gold/route.ts`                                           | Không có mã lỗi (`code`) riêng — dự án nhỏ, message string đủ dùng hiện tại.                                                                                                                                |
| Lỗi provider dữ liệu ngoài          | Class `ProviderError(provider, message, cause?)` — luôn gắn tên provider vào message                                                                                            | `lib/providers/types.ts`                                                                               | Dùng cho Twelve Data/Stooq; adapter BTMC domestic-gold parse phòng thủ, bỏ dòng hỏng thay vì throw toàn bộ (`lib/providers-domestic/btmc.ts`).                                                              |
| Nguồn dữ liệu API (`source`)        | Mọi API route trả `source: 'supabase' \| 'sample'` — UI luôn hiển thị banner cảnh báo khi `source === 'sample'`                                                                 | `app/api/candles/route.ts`, `app/api/domestic-gold/route.ts`                                           | Không được bỏ field này khi thêm route mới lấy dữ liệu từ Supabase.                                                                                                                                         |
| Check quyền / ghi dữ liệu           | Ghi (INSERT/UPDATE) chỉ qua `service_role` trong Edge Function; client luôn dùng anon key qua RLS, chỉ SELECT                                                                   | `supabase/migrations/*.sql`, `supabase/functions/*/index.ts`                                           | RLS có policy SELECT + `GRANT SELECT` tường minh cho `anon`/`authenticated` (bài học đã ghi ở PROGRESS.md — thiếu GRANT thì RLS không đủ).                                                                  |
| Trạng thái UI                       | Đủ 4 trạng thái: `loading` (`role="status" aria-live="polite"`) · rỗng (`role="status"`, viền nét đứt) · `error` (`role="alert"`) · `success`                                   | `app/chart/[symbol]/chart-page-client.tsx`, `app/gia-vang-trong-nuoc/page-client.tsx`                  | Hook fetch riêng mỗi feature (`use-candles.ts`, `use-domestic-gold.ts`) trả `{status, data, source, error}` cùng shape.                                                                                     |
| Thời gian                           | UTC nhất quán trong DB/API; quy đổi giờ địa phương (VN UTC+7) chỉ ở lớp adapter khi parse response ngoài                                                                        | `lib/providers-domestic/btmc.ts`                                                                       | Không lưu giờ địa phương vào DB.                                                                                                                                                                            |

| Đặt tên file/thư mục                | File `kebab-case.ts(x)`; export component `PascalCase`; hàm/biến `camelCase`; hằng số `UPPER_SNAKE_CASE`; bảng/cột DB `snake_case`                                              | `components/theme-toggle.tsx`, `lib/candles/types.ts`                                                  | Theo đúng CLAUDE.md mục 10.                                                                                                                                                                                 |
| Hook đọc localStorage/URL sau mount | State bắt đầu bằng default (khớp SSR) → đọc `localStorage`/URL THẬT sau `useEffect` (tránh lệch hydrate) → tự đồng bộ lại khi đổi                                               | `components/theme-toggle.tsx`, `components/chart/use-indicator-config.ts`                              | Đây là pattern bắt buộc cho mọi state phụ thuộc `window`/`localStorage` — không đọc trực tiếp lúc render.                                                                                                   |
